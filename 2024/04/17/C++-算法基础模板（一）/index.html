快速排序算法

对应的模板题–对应 785. 快速排序 - AcWing题库

测试了一下 使用stl里面的sort也是可以通过的 问题不大

快速排序的时间复杂度取决于数组中元素的分布情况。其复杂度如下：

  1. 最好情况（Best case）：如果每次分区操作都能将数组分成两个几乎相等的部分，快速排序的时间复杂度为 $O(n \log n)$。

  2. 平均情况（Average case）：对于随机排列的输入，快速排序的期望时间复杂度也是 $O(n \log n)$。

  3. 最坏情况（Worst case）：当数组已经是有序的（无论升序还是降序），或者每次分区选择的枢纽元是当前数组中的最小或最大元素时，快速排序的时间复杂度会退化到 $O(n^2)$。

快速排序的性能通常比其他 $O(n \log n)$ 排序算法更优，尤其是在平均情况下，但在最坏情况下可能会表现不佳。因此，一些快速排序的变体，如随机化快速排序，通过随机选择枢纽元来尝试避免最坏情况的发生，使得其性能更加稳定。

归并排序

对应的模板题–对应模板题787. 归并排序 - AcWing题库

归并排序（Merge Sort）是一种有效的排序算法，基于分治法的思想。其主要步骤包括将一个大数组分成两半，递归地对这两半分别进行排序，然后将它们合并在一起。这个过程涉及到多个层级的分割和合并。

归并排序的时间复杂度分析如下：

  1. 分割：每次递归调用都将数组分割成两个子数组，每个子数组大约是原数组大小的一半。这个分割过程会持续进行，直到子数组的大小缩减到1。

  2. 合并：在每一层递归中，都会对分割后的子数组进行合并。合并两个大小为 $ n/2 $ 的数组的时间复杂度是 $ O(n) $，因为每个元素最多被复制和比较一次。

  3. 递归层数：由于每次递归都将数组分成两半，所以递归的层数是 $ \log_2 n $。这是因为数组大小每层递归都减半，直到达到1。

综合上述分析，归并排序的总时间复杂度为每层的合并复杂度 $ O(n) $ 乘以层数 $ \log_2 n $，因此总的时间复杂度为 $ O(n \log n) $。

这个 $ O(n \log n) $ 的时间复杂度是在最好、平均和最坏情况下都一致的，这使得归并排序在不同的输入情况下表现稳定。

整数二分

对应的模板题目—789. 数的范围 - AcWing题库

对于下面的对于复杂度的描述可能是存在错误的

这两个二分搜索的函数 bsearch_1 和 bsearch_2 都涉及到对区间 [l, r] 的划分，并且每次迭代都将区间长度减半。关键在于每次迭代中调用 check 函数来决定怎样缩小搜索范围。

对于这两个函数的时间复杂度分析，我们可以从以下几个方面来看：

  1. 迭代次数：

    • 每次迭代中，搜索区间的大小大约减半。初始区间大小为 $r - l$，每次迭代后变为 $\frac{r - l}{2}$，然后是 $\frac{r - l}{4}$，依此类推。
    • 区间长度将持续减少直到 $l$ 与 $r$ 相遇，即 $l = r$。因此，迭代次数大约是 $\log_2(r - l + 1)$，即从初始区间到最小区间所需要的步骤数。

  2. 每次迭代的复杂度：

    • 在每次迭代中，唯一的计算是计算中点 mid，并调用一次 check(mid) 函数。
    • 假设 check 函数的时间复杂度为 $O(C)$，那么每次迭代的时间复杂度也为 $O(C)$。

因此，整个二分搜索的总时间复杂度是迭代次数乘以每次迭代的复杂度，即 $O(C \log_2(r - l + 1))$。

其中 bsearch_1 和 bsearch_2 的主要区别在于如何计算 mid 和如何调整边界 l 和 r，但这不影响总的时间复杂度。两者都符合对数时间复杂度，即假设 check 函数是常数时间操作，则二分搜索的总时间复杂度为 $O(\log_2(r - l + 1))$。如果 check 函数具有更高的复杂度 $O(C)$，则总复杂度为 $O(C \log_2(r - l + 1))$。
